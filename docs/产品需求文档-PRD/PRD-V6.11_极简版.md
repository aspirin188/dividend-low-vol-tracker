### Bug修复详情

#### 1. 分红年数计算逻辑错误

**v6.10 问题**：
```python
# 当前代码统计的是"累计年份数"
dividend_years.add(report_date.year)
return len(dividend_years)  # 返回 1-5
```

**问题示例**：
- 2020年分红、2022年分红、2024年分红 → 返回3年
- 但实际不是"连续"分红

**v6.11 修复**：
```python
# 改为统计"连续"分红年数
# 从最近一年往前推，连续有分红的年份数
# 如果中间断开，则从断开处重新计数
```

**正确逻辑**：
- 连续分红定义：从最近一个有分红的年份开始，往前连续有分红的年份数
- 例如：2024年有、2023年有、2022年无 → 返回2年（只计连续）
- 例如：2024年有、2023年有、2022年有 → 返回3年

#### 2. 分红判断条件过于粗糙

**v6.10 问题**：
```python
if desc and '派' in str(desc):  # 只检查"派"字
    dividend_years.add(report_date.year)
```

**问题**：
- "10送5股" → 没有现金分红，但有股票分红，被漏掉
- "10转增10股" → 没有现金分红，但有资本公积转增，被漏掉

**v6.11 修复**：
```python
# 判断是否有分红：包含"派"或"送"字样
if desc and ('派' in str(desc) or '送' in str(desc)):
    dividend_years.add(report_date.year)
```

**说明**：
- "派"：现金分红（如"10派3元"）
- "送"：股票分红（如"10送5股"）
- "转增"通常不算分红，是资本公积转增股本

### 新增筛选条件详情

#### ROE筛选

**数据来源**：东方财富数据中心 RPT_LICO_FN_CPD 接口
- 字段：WEIGHTEDAVERAGEORE（加权平均净资产收益率）

**筛选规则**：
```
ROE ≥ 8%

理由：
- 巴菲特标准：优秀公司ROE应长期保持15%以上
- 保守设置8%，确保基本盈利能力
- 剔除ROE过低或负值的公司
```

#### 资产负债率筛选

**数据来源**：东方财富数据中心 RPT_LICO_FN_CPD 接口
- 字段：ASSETLIABRATIO（资产负债率）

**筛选规则**：
```
一般行业：资产负债率 ≤ 70%
金融地产：资产负债率 ≤ 85%（行业特性）

理由：
- 高负债率意味着高风险
- 金融、地产行业负债率天然较高，需区别对待
- 剔除"高股息但高负债"的风险标的
```

## 3. 三因子评分模型（v6.10）

```
综合评分 = 股息率归一化 × 0.5 + (1 - 波动率归一化) × 0.3 + 分红稳定性归一化 × 0.2

数据口径:
- 股息率:TTM(近12个月,自计算)
- 波动率:120日对数收益率,年化 √242
- 分红稳定性:过去5年连续分红年数

硬性筛选(v6.11):
- 股息率 ≥ 3%
- 总市值 ≥ 500亿
- 股利支付率 ≤ 150%
- EPS > 0
- 非ST
- 连续分红年数 ≥ 3年（v6.11修复计算逻辑）
- ROE ≥ 8%（v6.11新增）
- 资产负债率 ≤ 70%（金融地产≤85%）（v6.11新增）
```
