## 4. 两因子评分模型

### 4.1 为什么是两因子？

```
传统四因子：股息率 + 波动率 + 分红质量 + 盈利稳定性

极简两因子：股息率 + 波动率

简化理由：
1. 分红质量 ← 本质是「历史股息率是否稳定」，被波动率部分代理
2. 盈利稳定性 ← 本质是「价格波动是否低」，直接等于波动率
3. 两因子已经抓住了「收益」和「风险」的核心
```

### 4.2 固定权重

```
权重固定为：股息率 60%，波动率 40%

用户不需要知道为什么，这就是默认的最佳配置。
```

### 4.3 评分算法

```python
def calculate_composite_score(stock, all_stocks) -> float:
    """
    极简两因子评分。

    公式：综合评分 = 股息率百分位 × 0.6 + (100 - 波动率百分位) × 0.4
    """

    # 股息率百分位（越高越好）
    dividend_yields = [s['dividend_yield'] for s in all_stocks]
    dividend_percentile = percentile_rank(dividend_yields, stock['dividend_yield'])

    # 波动率百分位（越低越好，取反）
    volatilities = [s['annual_vol'] for s in all_stocks]
    volatility_percentile = percentile_rank(volatilities, stock['annual_vol'])
    volatility_score = 100 - volatility_percentile

    # 综合评分
    composite = dividend_percentile * 0.6 + volatility_score * 0.4

    return round(composite, 2)


def percentile_rank(values: list[float], target: float) -> float:
    """计算百分位排名"""
    if not values or len(values) <= 1:
        return 50.0
    if all(v == values[0] for v in values):
        return 50.0

    less_than = sum(1 for v in values if v < target)
    equal_to = sum(1 for v in values if v == target)
    percentile = (less_than + 0.5 * equal_to) / len(values) * 100

    return round(percentile, 2)
```

### 4.4 硬性筛选条件

2. 总市值 ≥ 100亿

### 6.4 一键运行流程

```python
@app.route('/api/run', methods=['POST'])
def run():
    """
    一键运行：获取分红 → 获取行情 → 计算波动率 → 筛选 → 评分

    简单实现：同步执行，不使用SSE。
    数据量不大时约3-5分钟，用户可以等待。
    """
    # 1. 获取分红数据
    dividends = fetch_all_dividends()  # 约5秒

    # 2. 获取实时行情
    quotes = fetch_all_quotes()  # 约20秒

    # 3. 计算波动率
    volatilities = calculate_all_volatilities()  # 约3分钟

    # 4. 合并数据
    merged = merge_data(dividends, quotes, volatilities)

    # 5. 筛选
    filtered = filter_stocks(merged)

    # 6. 评分
    scored = calculate_scores(filtered)

    # 7. 保存结果
    save_results(scored)

    return jsonify({'success': True, 'count': len(scored)})
```
