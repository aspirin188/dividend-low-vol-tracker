## 详细修复方案

### F-01: 市值数据修复

**现状**: `result['market_cap'] = result['price'] * 100`（粗略估算）
**目标**: 获取真实市值

**方案**: 新浪实时行情接口 `hq.sinajs.cn` 返回的字段中：
- fields[0] = 股票名称
- fields[1] = 今日开盘价
- fields[2] = 昨日收盘价
- fields[3] = 当前价格
- fields[20] = 总股本（万）
- fields[21] = 流通股本（万）

市值 = 当前价格 × 总股本 / 10000（万元→亿元）

**修改位置**: `fetcher.py` → `fetch_realtime_prices()` 增加总股本字段
**验证方式**: 打印几只已知大盘股的市值验证（如工行 ~2.2万亿）

### F-02: 分红年数修复

**现状**: `result['dividend_years'] = 5`（固定值）
**目标**: 从数据源获取真实的连续分红年数

**方案**: 使用 `akshare.stock_fhps_em()` 批量获取多年分红数据（2021-2024），统计每只股票的分红年数。

**数据源**:
- `ak.stock_fhps_em(date='20241231')` — 2024年报
- `ak.stock_fhps_em(date='20231231')` — 2023年报
- `ak.stock_fhps_em(date='20221231')` — 2022年报
- `ak.stock_fhps_em(date='20211231')` — 2021年报

统计逻辑：一只股票在 4 年中有 N 年有分红，则 dividend_years = N。

**修改位置**: `fetcher.py` → 新增 `fetch_dividend_years_batch()` 函数
**验证方式**: 已知高稳定分红股（如工行、建行）应为 4 年

### F-03: 支付率稳定性评分修复

**现状**: `calculate_payout_stability_score()` 返回 `(None, 50.0)`
**目标**: 计算真实的 3 年平均支付率和稳定性

**方案**: 利用 F-02 已获取的多年分红数据，结合 EPS 计算：
- 支付率 = 每股分红 / 每股收益
- 稳定性 = 1 - (支付率标准差 / 支付率均值)

需要获取多年 EPS 数据：`akshare.stock_yjbb_em(date=...)`

**修改位置**: `fetcher.py` → 重写 `calculate_payout_stability_score()`
**验证方式**: 银行股支付率通常在 25-35%，稳定性应 >80 分

### F-04: PE/PB 修复

**现状**: `result['pe'] = None; result['pb'] = None`
**目标**: 获取真实 PE/PB

**方案**: 新浪实时行情接口字段：
- fields[39] = 市盈率（PE, 动态）
- fields[46] = 市净率（PB）

注意：部分股票可能没有 PE（亏损股）或 PB（资不抵债），需要处理 None。

**修改位置**: `fetcher.py` → `fetch_realtime_prices()` 增加字段
**验证方式**: 银行股 PE 约 4-6 倍，PB 约 0.5-0.7 倍

## v8.3 新增修复（2026-04-01）

### F-09: 信号逻辑矛盾修复

**问题描述**:
用户反馈"信号"和"趋势"相互矛盾，如趋势向上但信号却是"强制卖出"。

**问题根源**:
原代码中，死叉检测（MA20 < MA60）的优先级最高，不区分趋势方向直接触发"强制卖出"。这导致即使长期趋势向上，短期死叉也会触发强制卖出信号，与趋势矛盾。

**修复方案**:
根据趋势方向对死叉信号降级处理：
- 趋势向下 + 死叉 → 强制卖出（signal_level=-4）
- 横盘 + 死叉 → 减仓（signal_level=-2）
- **趋势向上 + 死叉 → 警示（signal_level=-1）** ⭐ 核心修复

**修复代码** (`fetcher.py:727-754`):
```python
# 死叉检测（MA20 < MA60）
if ma20 < ma60:
    if trend == '向下':
        signal = "强制卖出"
        signal_level = -4
    elif trend == '横盘':
        signal = "减仓"
        signal_level = -2
    else:  # trend == '向上'
        signal = "警示"
        signal_level = -1  # 降级处理
```

**测试结果**:
```

趋势分布: {'向上': 3, '横盘': 2}
信号分布: {'警示': 2, '减仓': 1, '试探买入': 1, '观望': 1}
矛盾案例（趋势向上+强制卖出）: 0 个 ✅

工商银行: 趋势向上 + 死叉 → 警示（正确降级）
农业银行: 趋势向上 + 死叉 → 警示（正确降级）
```

**验收**: ✅ 测试通过，无矛盾案例
