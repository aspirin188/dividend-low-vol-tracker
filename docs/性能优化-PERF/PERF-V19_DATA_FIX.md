# v6.19 数据完整性修复说明

## 📅 修复日期
2026-03-29 15:40

## 🐛 问题描述

用户反馈两个数据缺失问题：

### 1. 负债率部分缺失
- **现象**: 部分股票的负债率显示为空
- **影响**: 用户无法准确筛选负债率条件

### 2. 股价百分位全部缺失
- **现象**: 所有股票的股价百分位都显示为空
- **影响**: 用户无法使用股价百分位筛选功能

---

## 🔍 问题原因

查看代码发现，在 `server/services/fetcher.py` 中：

```python
# 问题代码（v6.19之前）
debt_results = calculate_debt_ratio_batch(candidate_codes[:20], ...)  # ❌ 只计算前20只
price_percentiles = calculate_price_percentile_batch(candidate_codes[:30], ...)  # ❌ 只计算前30只
```

**原因**：
- 为了"避免接口限制"，代码硬编码限制了计算数量
- 如果候选池有100只股票：
  - 只有前20只会有负债率数据
  - 只有前30只会有股价百分位数据
  - 其他股票这些字段为空

---

## ✅ 修复方案

### 代码修改

```python
# 修复后代码（v6.19）
debt_results = calculate_debt_ratio_batch(candidate_codes, delay=0.2, timeout=10)  # ✅ 计算所有候选股
price_percentiles = calculate_price_percentile_batch(candidate_codes, days=250)  # ✅ 计算所有候选股
```

### 修改点

1. **移除数量限制**
   - `candidate_codes[:20]` → `candidate_codes`
   - `candidate_codes[:30]` → `candidate_codes`

2. **优化延迟参数**
   - `delay=0.3` → `delay=0.2`
   - 平衡性能和接口限制

3. **更新日志输出**
   - 显示实际计算的股票数量

## ⚠️ 注意事项

### 性能影响

移除数量限制后，数据获取时间会增加：

| 操作 | 修复前 | 修复后 | 增加 |
|------|--------|--------|------|
| 负债率计算 | 20只 × 0.3s = 6秒 | 100只 × 0.2s = 20秒 | +14秒 |
| 股价百分位计算 | 30只 | 100只 | +70只 |

**总耗时增加**: 约20-30秒

### 接口限制风险

- **负债率**: 需要逐个股票获取财务数据，可能触发频率限制
- **股价百分位**: 需要获取历史价格数据，可能触发频率限制

**缓解措施**:
- 延迟参数设置为0.2秒（平衡速度和稳定性）
- 如果遇到频繁失败，可适当增加延迟（如0.3秒）
