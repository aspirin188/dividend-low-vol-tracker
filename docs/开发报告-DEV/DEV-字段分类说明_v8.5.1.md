# 字段分类说明 - v8.5.1 优化

## 核心理念

**静态字段**（和股票绑定）：配置改变时不变，应该缓存在 `stock_raw_data` 表中
**动态字段**（依赖配置）：配置改变时需要重新计算，存储在 `stock_data` 表中

这样可以节省 **60-100倍** 的时间！

---

## 一、静态字段（和股票绑定）

这些字段是股票的客观属性，不受配置影响，只需获取一次：

### 1. 基础信息
- `code` - 股票代码
- `name` - 股票名称
- `industry` - 所属行业
- `market` - 市场类型（沪市/深市/创业板/科创板）

### 2. 行情数据
- `price` - 当前股价
- `pe` - 市盈率
- `pb` - 市净率
- `market_cap` - 总市值
- `eps` / `basic_eps` - 每股收益

### 3. 分红数据
- `dividend_yield` / `dividend_yield_ttm` - 股息率
- `dividend` - 分红金额
- `div_per_share` - 每股分红
- `payout_ratio` - 股利支付率
- `payout_3y_avg` - 3年平均支付率
- `dividend_years` - 连续分红年数

### 4. 财务指标
- `roe` - 净资产收益率
- `debt_ratio` - 资产负债率
- `profit_growth_3y` - 近3年净利润增长率
- `roe_trend` - ROE趋势变化
- `peg` - PEG指标（市盈率相对盈利增长比率）

### 5. 波动率指标
- `annual_vol` - 年化波动率
- `price_percentile` - 股价历史百分位

### 6. 技术指标（均线系统）
- `ma250` - 250日均线
- `ma20` - 20日均线
- `ma60` - 60日均线
- `current_price` - 当前价格
- `price_vs_ma_pct` - 价格相对250日均线位置
- `ma_slope` - 均线斜率
- `trend` - 趋势方向（上涨/下跌/横盘）
- `trend_strength` - 趋势强度
- `signal` - 信号描述（如"强烈买入"）
- `signal_level` - 信号级别（1-5级，负数为卖出信号）
- `signal_type` - 信号类型（buy/sell/hold）
- `action` - 操作建议

### 7. 质量因子
- `cashflow_profit_ratio` - 现金流质量比率
- `top1_shareholder_ratio` - 第一大股东持股比例

### 8. 元数据
- `pinyin_abbr` - 拼音首字母缩写
- `data_date` - 数据日期
- `updated_at` - 更新时间
- `fetch_time` - 获取耗时

## 二、动态字段（依赖配置）

这些字段会根据配置参数变化，需要重新计算：

### 1. 筛选结果
**受影响参数：**
- `MIN_DIVIDEND_YIELD` / `MAX_DIVIDEND_YIELD` - 股息率范围
- `MIN_MARKET_CAP` - 最小市值
- `MIN_ROE` - 最小ROE
- `MAX_DEBT_RATIO` - 最大负债率
- `MIN_DIVIDEND_YEARS` - 最小分红年数
- `ENABLE_PROFIT_GROWTH_FILTER` - 是否启用利润增长筛选
- `MIN_PROFIT_GROWTH` - 最小利润增速

**结果：** 某只股票是否通过筛选会变化

### 2. 综合评分
**评分公式：**
```
综合评分 = 股息率权重 × 股息率得分 
         + 波动率权重 × 波动率得分 
         + 稳定性权重 × 稳定性得分 
         + 成长权重 × 成长因子得分
```

**受影响参数：**
- `WEIGHT_DIVIDEND` - 股息率权重
- `WEIGHT_VOL` - 波动率权重
- `WEIGHT_STABILITY` - 稳定性权重
- `WEIGHT_GROWTH` - 成长权重

**结果：** 不同权重下，评分会不同

### 3. 排名
**计算方式：** 根据综合评分降序排列

**受影响：** 评分变化 → 排名变化

### 4. 击球区评分
**评分模型：**
```
击球区评分 = 价格百分位得分(0-20) 
           + 估值得分(0-20) 
           + 均线位置得分(0-20)
```

**注意：** 虽然击球区评分的计算逻辑固定，但它依赖于筛选后的股票池，所以会间接受配置影响。

## 三、优化效果对比

### 传统方式（每次配置改变都重新获取）
```
完整运行流程：
1. 获取行情数据（30秒）
2. 获取分红数据（30秒）
3. 获取财务数据（60秒）
4. 获取技术指标（60秒）
5. 计算均线系统（30秒）
6. 筛选（5秒）
7. 评分（5秒）
总计：约3-5分钟
```

### 优化后（只重算动态部分）
```
快速重筛流程：
1. 从缓存表读取静态数据（1秒）
2. 筛选（2秒）
3. 评分（2秒）
总计：<5秒
```

**性能提升：60-100倍！**

## 四、使用场景

### 场景1：首次运行
```python
POST /api/run  # 完整运行，获取所有数据
```
- 获取所有静态字段
- 保存到 `stock_raw_data` 表
- 执行筛选和评分
- 保存结果到 `stock_data` 表

### 场景2：配置改变后（推荐）
```python
POST /api/rescreen  # 快速重筛，只重算动态部分
```
- 从 `stock_raw_data` 读取静态数据（1秒）
- 根据新配置重新筛选（2秒）
- 根据新权重重新评分（2秒）
- 保存结果到 `stock_data` 表

### 场景3：数据过期
```python
POST /api/run  # 完整运行，重新获取数据
```
- 检测到数据日期不是今天
- 需要重新获取静态数据

## 五、技术实现

### 数据表结构

**stock_raw_data（静态数据表）**
```sql
CREATE TABLE stock_raw_data (
    code TEXT PRIMARY KEY,
    name TEXT,
    industry TEXT,
    -- ... 所有静态字段
    fetch_time REAL,      -- 获取耗时
    updated_at TEXT       -- 更新时间
)
```

**stock_data（动态结果表）**
```sql
CREATE TABLE stock_data (
    code TEXT PRIMARY KEY,
    name TEXT,
    -- ... 筛选和评分结果
    composite_score REAL,  -- 综合评分
    rank INTEGER,          -- 排名
    updated_at TEXT        -- 更新时间
)
```

### API端点

**完整运行：** `/api/run`
- 获取所有数据（静态+动态）
- 耗时：3-5分钟

**快速重筛：** `/api/rescreen`
- 只重算动态部分
- 耗时：<5秒

**数据状态：** `/api/status`
- 检查静态数据是否存在
- 检查数据日期是否过期
- 返回是否需要刷新

## 六、最佳实践

### ✅ 推荐做法
1. 每天首次运行使用 `/api/run`（获取最新数据）
2. 配置调整后使用 `/api/rescreen`（快速响应）
3. 切换策略时使用 `/api/rescreen`（节省时间）

### ❌ 避免做法
1. 每次配置改变都调用 `/api/run`（浪费时间）
2. 数据过期后仍使用 `/api/rescreen`（会报错）
3. 首次运行使用 `/api/rescreen`（没有静态数据）

## 七、常见问题

### Q1: 为什么信号和技术指标是静态的？
**A:** 信号是基于价格计算的，配置改变不影响历史价格，所以信号不变。

### Q2: 为什么成长因子得分是静态的？
**A:** `profit_growth_3y`、`roe_trend` 等是客观财务数据，不会因配置改变而变化。

### Q3: 什么时候需要重新运行 `/api/run`？
**A:** 
- 每天首次打开系统
- 数据日期过期
- 需要获取最新行情数据

### Q4: 如何判断数据是否过期？
**A:** 
```python
GET /api/status
# 返回：
{
  "raw_data": {
    "data_date": "2026-04-01",
    "is_today": true/false
  },
  "need_fetch": false/true
}
```
