# 性能优化实施报告 v8.5

## 优化内容

### 1. 数据库架构优化

**新增表结构**：
- `stock_raw_data`：存储静态数据（与配置无关）
- `stock_data`：存储筛选结果（依赖配置）

**静态数据（stock_raw_data）包含**：
- 基础数据：价格、市值、PE、PB、股息率、波动率
- 财务数据：ROE、负债率、EPS、分红年数、支付率
- 质量因子：利润增速、ROE趋势、PEG、现金流质量
- 技术指标：均线位置、趋势、信号级别
- 元数据：数据日期、获取耗时

**筛选结果（stock_data）包含**：
- 筛选后的股票列表
- 综合评分、排名
- 动态计算结果

### 2. 新增API接口

#### 2.1 `/api/rescreen` - 快速重筛

**功能**：基于已有的静态数据进行筛选和评分

**优势**：
- 响应速度快（<5秒）
- 不需要重新获取数据
- 适合配置变化后快速查看结果

**流程**：
```
1. 检查stock_raw_data表中是否有今天的数据
2. 如果有，读取静态数据
3. 根据当前配置重新筛选
4. 重新计算评分和排名
5. 保存到stock_data表
6. 返回结果（耗时<5秒）
```

**返回示例**：
```json
{
  "success": true,
  "count": 81,
  "data_date": "2026-04-01",
  "screen_time": 2.3,
  "message": "快速重筛完成（耗时2.3秒）"
}
```

#### 2.2 `/api/status` - 数据状态检查

**功能**：检查静态数据和筛选结果的状态

**返回示例**：
```json
{
  "success": true,
  "raw_data": {
    "count": 5000,
    "data_date": "2026-04-01",
    "fetch_time": 180.5,
    "is_today": true
  },
  "screened_data": {
    "count": 81,
    "data_date": "2026-04-01"
  },
  "need_fetch": false
}
```

#### 2.3 `/api/run` - 完整运行（优化版）

**新增功能**：
- 获取数据后保存到`stock_raw_data`表（静态数据）
- 筛选后保存到`stock_data`表（筛选结果）
- 返回更多信息（静态数据行数、完整/快速标记）

### 3. 前端优化

#### 3.1 配置页面优化

**优化前流程**：
```
点击策略 → 应用配置 → 跳转首页 → 点击运行 → 等待3-5分钟

**优化后流程**：
```

  └─ 无今天的静态数据 → 提示用户 → 完整获取（3-5分钟）

**用户体验改进**：
- 策略切换响应速度提升60-100倍（从3-5分钟降至<5秒）
- 智能判断是否需要重新获取数据

## 技术实现细节

### 1. 数据分离原则

**静态数据（不随配置变化）**：
- 价格、市值、PE、PB
- 股息率、波动率、分红年数
- ROE、负债率、EPS
- 均线位置、趋势、信号
- 利润增速、ROE趋势

**动态数据（随配置变化）**：
- 筛选结果（哪些股票入选）
- 综合评分（权重可变）
- 排名

### 2. 数据一致性保证

**数据版本控制**：
- 每条静态数据记录`data_date`和`updated_at`
- 筛选前检查`data_date`是否为今天
- 如果过期，提示用户刷新

**并发控制**：
- 快速重筛时读取静态数据（只读）
- 筛选结果写入`stock_data`表（清空+插入）
- 避免数据竞争

### 3. 错误处理

**情况1：静态数据不存在**
```json
{
  "success": false,
  "error": "没有静态数据，请先运行完整获取",
  "need_fetch": true
}
```

**情况2：静态数据已过期**
```json
{
  "success": false,
  "error": "静态数据已过期（2026-03-31），请先刷新数据",
  "need_fetch": true,
  "data_date": "2026-03-31"
}
```

**情况3：筛选无结果**
```json
{
  "success": false,
  "error": "没有符合条件的股票"
}
```
