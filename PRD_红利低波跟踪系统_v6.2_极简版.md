## 4. 两因子评分模型

### 4.1 为什么是两因子？

```
传统四因子：股息率 + 波动率 + 分红质量 + 盈利稳定性

极简两因子：股息率 + 波动率

简化理由：
1. 分红质量 ← 本质是「历史股息率是否稳定」，被波动率部分代理
2. 盈利稳定性 ← 本质是「价格波动是否低」，直接等于波动率
3. 两因子已经抓住了「收益」和「风险」的核心
```

### 4.2 数据口径定义

```
┌──────────────────────────────────────────────────────┐
│              数据口径 — 写死，不可配置                   │
├──────────────────────────────────────────────────────┤
│  股息率：TTM（Trailing Twelve Months，近12个月）       │
│          = 最近一次每股分红 × 年分红次数 / 当前股价     │
│                                                      │
│  波动率：120个交易日对数收益率，年化                    │
│          = STD(ln(Pt/Pt-1)) × √242                   │
│          其中 √242 为A股年交易日数                     │
│                                                      │
│  总市值：使用最新实时行情的总市值（亿元）               │
│                                                      │
│  股利支付率：= 每股分红 / 每股收益(EPS) × 100%        │
│              若 EPS ≤ 0，直接过滤                      │
│                                                      │
│  股价：使用 push2 接口实时行情 f2 字段（最新价）       │
│                                                      │
│  市盈率(PE)：使用 push2 接口 f9 字段（动态市盈率）     │
│             = 总市值 / 净利润(TTM)                    │
│                                                      │
│  市净率(PB)：使用 push2 接口 f23 字段                  │
│             = 总市值 / 净资产                         │
└──────────────────────────────────────────────────────┘
```

### 4.3 固定权重

```
权重固定为：股息率 60%，波动率 40%

用户不需要知道为什么，这就是默认的最佳配置。
```

### 4.4 评分算法

```python
def calculate_composite_score(stock, all_stocks) -> float:
    """
    极简两因子评分（min-max 归一化）。

    公式：综合评分 = 股息率归一化 × 0.6 + (1 - 波动率归一化) × 0.4
    """

    # 股息率归一化（越高越好）
    dividend_yields = [s['dividend_yield'] for s in all_stocks]
    dividend_norm = min_max_normalize(dividend_yields, stock['dividend_yield'])

    # 波动率归一化（越低越好，取反）
    volatilities = [s['annual_vol'] for s in all_stocks]
    vol_norm = min_max_normalize(volatilities, stock['annual_vol'])
    vol_score = 1.0 - vol_norm

    # 综合评分（映射到 0-100 分）
    composite = (dividend_norm * 0.6 + vol_score * 0.4) * 100

    return round(composite, 2)


def min_max_normalize(values: list[float], target: float) -> float:
    """
    Min-Max 归一化，将值映射到 [0, 1]。

    边界情况：
    - 空列表或单个元素：返回 0.5
    - 所有值相同：返回 0.5
    """
    if not values or len(values) <= 1:
        return 0.5

    min_val = min(values)
    max_val = max(values)

    if min_val == max_val:
        return 0.5

    return (target - min_val) / (max_val - min_val)
```

### 4.5 硬性筛选条件

```
保留以下全部通过的公司（任一不满足则排除）：
1. 股息率(TTM) ≥ 4.0%
2. 总市值 ≥ 100亿
3. 股利支付率 ≤ 150%（且 EPS > 0）
4. 当年 EPS > 0
5. 非ST股票
6. 数据完整性：股息率、波动率、市值、EPS 四个字段均非空
```

### 4.6 缺失数据处理

```
原则：宁可少，不可错。

- 停牌股票：无行情数据，波动率无法计算 → 过滤
- 次新股：上市不足120个交易日，波动率不具代表性 → 过滤
- 分红数据缺失：股息率为空 → 过滤
- 任何关键字段为空或异常值（如负股息率、负市值） → 过滤
```

### 6.4 一键运行流程

```python
@app.route('/api/run', methods=['POST'])
def run():
    """
    一键运行：获取分红 → 获取行情 → 计算波动率 → 筛选 → 评分

    v6.2 更新：行情接口增加 f2(股价)、f9(PE)、f23(PB) 字段获取。
    """
    # 1. 获取分红数据
    dividends = fetch_all_dividends()

    # 2. 获取实时行情（含股价、PE、PB）
    quotes = fetch_all_quotes()  # 新增 f2, f9, f23

    # 3. 计算波动率（120个交易日对数收益率，年化√242）
    volatilities = calculate_all_volatilities()

    # 4. 合并数据
    merged = merge_data(dividends, quotes, volatilities)

    # 5. 筛选（硬性条件 + 数据完整性检查）
    filtered = filter_stocks(merged)

    # 6. 评分（min-max 归一化）
    scored = calculate_scores(filtered)

    # 7. 保存结果（含 data_date + price/pe/pb）
    save_results(scored)

    return jsonify({'success': True, 'count': len(scored)})
```

## 8. 验收标准

### 8.1 功能验收

| 验收项 | 标准 |
|--------|------|
| 一键运行 | 点击后自动完成全部流程，约3-5分钟，期间按钮禁用 |
| 标的列表 | 显示所有符合条件的股票，按评分降序，含股价/PE/PB/EPS列 |
| 搜索 | 输入名称或代码能快速定位 |
| 行业筛选 | 下拉选择行业后列表即时过滤 |
| 市值筛选 | 下拉选择市值区间后列表即时过滤 |
| 股息率筛选 | 下拉选择股息率下限后列表即时过滤 |
| 导出 | 下载xlsx文件，含13列数据，格式规范 |
| 数据过期提示 | 数据超过7天时显示黄色提醒 |
| 错误处理 | 运行失败时显示具体错误信息 |

### 8.2 性能验收

| 验收项 | 标准 |
|--------|------|
| 首次加载 | < 1秒 |
| 搜索响应 | < 100ms |
| 筛选响应 | < 100ms |
| 数据更新 | 3-5分钟完成全流程 |
| 超时不断 | 10分钟内不中断 |

### 8.3 极简验收

| 验收项 | 标准 |
|--------|------|
| 代码行数 | app.py < 100行，总代码 < 2000行 |
| 页面数量 | 只需要1个HTML页面 |
| 配置项 | 用户可见的配置为0 |

### 8.4 数据质量验收

| 验收项 | 标准 |
|--------|------|
| 股息率口径 | TTM（近12个月），与同花顺/东方财富可交叉验证 |
| 波动率计算 | 120日对数收益率年化（√242），可复现 |
| PE/PB口径 | 与东方财富 push2 接口一致（动态值） |
| 数据完整性 | 列表中无空值、无异常值（负股息率、零市值等） |
| 结果数量 | 符合条件标的在 50-300 只范围内 |
